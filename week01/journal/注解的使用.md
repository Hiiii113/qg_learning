# 注解的使用

在实际编写中，我们可能需要给某一个类、方法、成员函数上面加上注解

比如用于标识==方法重写==的`@override`等等

这里主要记录**如何自定义注解**以及**如何使用**

## 1. 注解的创建

在 idea 里面，可以直接选择创建一个注解

这里给出一个简单的注解示例代码：

```java
@Retention(RetentionPolicy.RUNTIME)  
@Target(ElementType.PARAMETER)  
public @interface Param  
{  
    String value();  
}
```

## 2. 注解的属性

注解其实就类似于一个说明书，我们可以在类、方法、成员函数上面加上注解，然后来获取相关注解的信息

其中注解有两个属性

- 可以加在哪？（Target）
- 存在多久？（Retention）

以及一个用于读取注解的方法：Reflection（反射）

### 2.1 Target

Target 属性是用于指定这个注解可以贴在哪里的，如果不指定，默认可以贴在任何地方

Target 的设置示例代码如下：

```java
@Target({ElementType.METHOD, ElementType.FIELD})
```

其中有几个**常用值**：

- `ElementType.TYPE`：类、接口、注解、枚举
- `ElementType.METHOD`：方法
- `ElementType.FIELD`：成员变量（属性）
- `ElementType.PARAMETER`：方法的参数

通过设定以上几个值，就可以指定这个注解可以贴的位置

### 2.2 Retention

Retention 这个属性决定了这个注解的存活周期

写法：

```java
@Retention(RetentionPolicy.RUNTIME)
```

Retention 有三种级别：
1. `RetentionPolicy.SOURCE`：**编译期丢弃**，只用于给编译期看（比如`@Override`）
2. `RetentionPolicy.CLASS`：**类加载时丢弃**，可以存在于 `.class` 文件中，但是不会被加载到内存
3. `RetentionPolicy.RUNTIME`：**运行期保留**，这是**框架**唯一能读取到的级别，如果想通过反射读取，必须选这个

### 2.3 存放与获取

注解不仅仅是给类、方法、成员函数带上一个标签，更重要的是，注解可以存放一些成员变量，可以让调用者获取

```java
@Table(name = "user_table") // 自定义注解 public class User
{
	private String name;
}
```

像这样，在注解定义的时候如果有**成员变量**，就可以在注解后的括号内**赋值**

我们可以使用**反射**（Reflection）来获取

```java
public class ReflectionDemo
{
    public static void main(String[] args) throws Exception
    {
        // 1. 获取 Class 对象
        Class<User> clazz = User.class;

        // 2. 检查类上有没有 @Table 注解
        if (clazz.isAnnotationPresent(Table.class))
        {
            // 3. 获取该注解对象
            Table table = clazz.getAnnotation(Table.class);
            
            // 4. 读取注解里的属性值（这就是反射的魔力）
            String tableName = table.name();
            System.out.println("成功读到数据库表名: " + tableName);
        }
    }
}
```

通过这样，我们就获取了一个注解中包含的东西
