# 正则表达式

**正则表达式是一种用于匹配所需字符串的计算机常用方法**，是一种用特殊符号描述文本规则的工具

> **简单说：用一串特殊符号来定义"我要匹配什么样的字符串"**

相关匹配规则如下：
- 限定符
- 或运算符
- 元字符
- 字符类
- 贪婪与懒惰匹配

## 1. 限定符

- `?` 限定符是限定符中常用的一种，表示它前面的字符可以出现 1 次或 0 次

- `*` 限定符的作用是：前面的字符可以出现 0 次或无数次

- `+` 限定符的作用是：前面的字符可以出现 1 次或无穷次，也就是对 `*` 的结果筛选出了至少出现了 1 次的情况

- `{...}` 限定符的作用：前面的字符可以出现`{}`中数字的次数，比如 `/ab{3}c` 就会匹配到 `abbbc`

还有更加高级的用法：

- `/ab{2,6}c` 会匹配到 b 的个数是 2-6 之间的
- `/ab{2,}c` 会匹配到 b 的个数是 2 个以上的

> 上面对于限定符的例子都是针对一个字符使用的，但是限定符还可以对多个字符使用，只需要给需要的字符套上括号即可

## 3. 元字符

元字符是正则表达式自带的一些定义好的匹配类型，不需要手动 `[0-9]` 来匹配数字

如下：

- `\d` 数字字符
- `\w` 单词字符（所有英文字母 + 数字 + 下划线 `_`）
- `\s` 空白字符（Tab + 换行符）
- `\D` 非数字字符
- `\W` 非单词字符
- `\S` 非空白字符
- `.` 所有字符（不包含 Tab）

除此以外，还有 `^` 和 `$`，他们分别表示匹配行首和行尾

比如：`/^a` 表示匹配处于行首的 `a`，`/a$` 表示匹配处于行尾的 `a`

## 4. 字符类

字符类表示的是：匹配到的结果中，用到的字符只能来自于这里面

例如：`[abc]`

也可以指定范围

比如 `[a-z]` 代表小写字母，`[a-zA-Z]` 代表所有字母，`[a-zA-Z0-9]` 代表所有字母和数字

其中我们可以使用 `^` 表示除了方括号内都可以匹配上

比如：`[^0-9]` 代表了所有非数字

## 5. 贪婪与懒惰匹配

这是一串 html 标签：`<span<b>this is an text</b>></span>`

当我们尝试使用正则表达式匹配的时候，可能会使用：`<.+>` 来处理

但是我们发现：这个正则表达式把所有的字段全部匹配了

**这是因为：`.` 代表任意字符，而 `+` 会匹配尽可能多的字符，导致了这样的结果**

解决方式也很简单：改成 `<.+?>` 即可将默认的**贪婪匹配**变成**懒惰匹配**
