# 鉴权机制

鉴权：判断用户是否有访问接口的权限

>例子：
>
>用户未登录时，无法访问登录后才能访问的接口
>
>普通用户，无法访问管理员才能访问的接口

在这些情况下，我们都需要判断用户是否具有相关的**权限**，从而保护服务器和数据库的安全

其中，相关的技术有：Cookie，Session，Token 三种

## 1. HTTP 协议深入了解

HTTP 协议：是超文本传输协议，主要用于浏览器和服务器之间交互数据，其中交互分为**请求**和**响应**

### 1.1 请求与响应

- **请求：**

```http
POST /api/login HTTP/1.1          ← ① 请求行
Host: localhost:8080              ← ② 请求头
Content-Type: application/json
User-Agent: PostmanRuntime
                                  ← ③ 空行（分隔头和体）
{"username":"admin","password":"123456"}   ← ④ 请求体请求行
```

- 其中请求头中常见的属性如下：

```http
Content-Type: 请求的报文形式
Accept: 客户端接收的数据类型
User-Agent: 发送请求的客户端类型
X-Requested-with: Ajax 请求，异步请求
Cookie: 请求携带的 Cookie 信息
```

- **响应：**

```http
HTTP/1.1 200 OK                          ← ① 状态行
Content-Type: application/json           ← ② 响应头
Server: Apache-Coyote/1.1
Date: Sun, 22 Mar 2026 10:30:00 GMT
                                         ← ③ 空行
{"code":200,"message":"登录成功","data":{"token":"abc123"}}   ← ④ 响应体
```

- 重要的响应头：

```http
Set-Cookie: 服务器返回的 Cookie 信息
```

### 1.2 HTTP 协议的特点

特点如下：

- 简单快捷
- 无连接无状态
  - 无连接无状态的意思是：**多个请求之间是没有关联的，相互独立**

这种**无连接无状态**的形式就导致了开发麻烦，因此就有了 Cookie 这个技术

## 2. Cookie 以及 Cookie 鉴权原理

想要了解 Cookie 是怎么鉴权的，首先需要明白 Cookie 是什么

### 2.1 什么是 Cookie ？

Cookie 是服务器产生的，保存在客户端的一小段文本信息，格式是`key:value`，也就是**键值对**

Cookie 分为两类：

- 会话级 Cookie：保存在浏览器内存中，关闭会话后 Cookie 会自动丢失
- 持久化 Cookie：保存在硬盘中，只有到达了失效时间才会失效

### 2.2 怎么查看 Cookie ？

既然 Cookie 在日常的网页使用中经常存在，那么 Cookie 如何查看呢？

步骤：点开网页，F12 打开开发者工具，选择 application（应用程序），找到 Cookie 选项并选择一个链接即可

![bilibili 的 Cookie](D:\repo\计算机\后端\assets\image-20260322153205092-1774164726570-31.png)

名词解释：

- 名称：Cookie 的名称
- 值：Cookie 携带的值
- Domain：范围，访问这个域名就会携带这个 Cookie
- Path：路径，访问这个路径就会携带这个 Cookie
- Expires / Max-Age：存活时间
  - 会话代表**会话 Cookie**，关闭会话后失效
  - 否则是截止日期，过期后才失效
- 大小：Cookie 的大小

> 除了直接在开发者工具查看，也可以在浏览器设置找到 Cookie 相关选项查看所有的 Cookie

### 2.3 Cookie 鉴权的原理

Cookie 鉴权的步骤：

1. 第一次访问网站时，服务器通过响应头里面的 Set-Cookie 字段给客户端响应 Cookie 并保存
2. 以后访问网站时，请求中的请求头都会带上这些 Cookie，实现鉴权

### 2.4 Cookie 的缺点

>  需要注意的是：Cookie 由于可以被直接查看，所以存储的都是一些不重要的数据

## 3. Session 以及 Session 鉴权的原理

Session 也是保存在浏览器的一个字符串，并且是通过 Cookie 实现的

### 3.1 Session 的鉴权原理

Session 在浏览器保存的步骤和 Cookie 很类似：

1. 第一次登录后，服务器通过 Set-Cookie 这个字段返回 SessionId，并且通过 Cookie 保存在浏览器
2. 后续操作时，同样会携带这个 SessionId 在请求头中，实现鉴权

> 需要注意：Session 在 Cookie 中保存都是**会话级别**的，默认的生命周期是**半个小时**

### 3.2 Session 的缺点

SessionId 同时保存在服务器中，非常占用服务器的内存

## 4. Token 以及 Token 的鉴权原理

### 4.1 Token 的鉴权原理

1. 登录后，服务器生成 Token 或者通过独立的接口生成 Token，然后存储在服务器的硬盘或者数据库里
2. 以后的操作带来的请求，必须在请求头或者请求参数里面带上 Token 实现鉴权

> Token 一般是经过加密，并且具有生命周期的，一般有两类：
>
> - access_token：生命周期在15分钟到2小时之间
> - refresh_token：生命周期是15天
